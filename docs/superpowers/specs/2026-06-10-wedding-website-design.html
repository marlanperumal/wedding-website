Overview

A wedding website for Marlan Perumal and Tramaine Liedeman. South Indian traditional aesthetic with a modern touch. Guests receive personalised invite links; the site handles digital invitations and per-person RSVPs across five events over two days in Cape Town, November 2026.

Dietary Options

Fixed set: Vegetarian, Vegan, Halal, Gluten-free, Nut allergy, Other (triggers free-text dietaryNotes field).

Invite Slug Format

[name-slug]-[4-char random]  e.g. naidoo-family-x3f2
Name slug generated from label on creation (admin-editable). Suffix is cryptographically random. Stored as single unique slug column.

Routing

RouteDescription
/[inviteSlug]Invite landing. Validates slug, sets guestInviteId cookie, renders personalised invite. 404 if unknown.
/rsvpRSVP form. Middleware-protected — redirects to / if no cookie.
/rsvp/confirmedPost-submission. Shows full venue addresses + .ics download links.
/adminPassphrase login page.
/admin/dashboardRSVP counts per event, dietary totals. Protected.
/admin/guestsGuest/invite management — add, edit, copy links. Protected.

Middleware

  • /rsvp* — no guestInviteId cookie → redirect to /
  • /admin/dashboard*, /admin/guests* — no valid adminSession → redirect to /admin

Key Flows

Invite Flow

1
Guest visits wedding.liedeman.perumal.co.za/naidoo-family-x3f2
2
Server Component looks up Invite by slug — 404 if not found
3
Route Handler sets signed HttpOnly guestInviteId=<invite.id> cookie (90-day expiry)
4
Renders invite page: personalised greeting, invited events grouped by day, venue names (not addresses), RSVP CTA

RSVP Flow

1
Guest navigates to /rsvp; middleware verifies guestInviteId cookie
2
Server Component loads Invite → guests + InviteEvent records + any existing Rsvp rows (supports editing)
3
Renders one card per Guest — only their invited events, grouped Thu / Fri
4
Guest confirms attendance per event, dietary requirements per person, contact email
5
Submit → Server Action → Zod validation → upsert Rsvp rows → mark Invite.submitted → send confirmation email → redirect to /rsvp/confirmed
6
/rsvp/confirmed reveals full venue addresses and .ics calendar links per event

Admin Flow

1
Passphrase form → bcrypt compare → set signed adminSession cookie → redirect to /admin/dashboard
2
Dashboard: RSVP counts per event (attending / not / no response), dietary breakdown totals
3
Guests: invite table with RSVP status; expandable rows for per-person detail; Add / Edit / Copy Link / Regenerate Slug actions

Local Dev & Deployment

Local Setup

docker-compose up -d        # start local Postgres
npm install
npx prisma db push          # sync schema
npx prisma db seed          # seed events + test invite
npm run dev

Seed creates 5 events and one test invite test-invite-dev1 with 2 guests, all events.

Deployment

  • Push to main → Vercel auto-deploys
  • prisma migrate deploy runs in Vercel build step
  • dev branch → Vercel preview deployment with separate preview DB
  • DNS at domains.co.za: CNAME wedding.liedeman under perumal.co.za → cname.vercel-dns.com

DNS note: liedeman.perumal.co.za is a second-level subdomain. You may need to add a CNAME for the full label wedding.liedeman under perumal.co.za — check what your domains.co.za control panel supports for nested subdomain CNAMEs.

Future Expansion (out of scope for v1)
  • Venue detail pages — add content to Event records
  • Wedding registry — new page, no DB changes
  • Gallery & photo upload — new Photo model, Vercel Blob storage
  • Additional info pages — static Next.js pages
